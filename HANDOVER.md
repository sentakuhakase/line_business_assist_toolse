# LINE Rich Menu Manager - 引き継ぎ資料

## 1. プロジェクト概要
LINE Messaging APIを利用して、エンジニア以外でもGUI上で簡単にリッチメニューの作成・画像紐付け・デフォルト設定管理を行えるWebツール。

## 2. 技術スタック
- **Frontend**: HTML5, CSS3 (Vanilla CSS), jQuery 3.6.0
- **Backend**: PHP (cURLによるAPIプロキシ)
- **認証**: サーバーにトークンを残さない `localStorage` 方式（ブラウザ保存）

## 3. 現在のファイル構成
- `index.php`: メインUI。サイドバーによる機能切り替え、モーダル管理。
- `api.php`: LINE APIへのリクエストを中継（CORS回避とステートレス処理）。
- `js/app.js`: フロントエンドの全ロジック。テンプレート定義、座標計算、プレビュー制御、バリデーション。
- `css/style.css`: デザイン定義。
- `img/richi_menu_official_temp/`: LINE公式のテンプレートガイド画像（Large/Compact, 各3解像度）。

## 4. 実装済み機能
- **リッチメニュー一覧**: 画像付きリスト表示。現在の「デフォルト」設定をバッジで可視化。
- **新規作成 (新フロー)**:
  1. テンプレート選択（LINE標準レイアウト全種対応）。
  2. 制作ガイド画像のダウンロード。
  3. デザイン済み画像のアップロード（作成前にプレビュー）。
  4. プレビュー画像への「エリアラベル (A, B, C...)」のオーバーレイ表示。
  5. 各エリアのアクション（URL, Message, Postback, Datetimepicker）の詳細設定。
  6. 「枠の作成」と「画像のアップロード」をワンクリックで連続実行。
- **デフォルト管理**: アカウント全体のデフォルトリッチメニューを設定・解除。
- **JSON表示**: 作成済みメニューの生データをコピー可能。

## 5. 重要な仕様・注意点
- **画像の上書き禁止**: LINE APIの仕様上、画像が紐付いたリッチメニューの画像のみを変更することはできません。変更が必要な場合は「削除して再作成」するフローとなります。
- **座標計算**: 内部では2500pxを基準とした比率で保持し、選択された解像度（Large/Medium/Small）に合わせて動的にスケーリングしてAPIへ送信しています。

## 6. 今後の課題（未実装機能）
- **特定ユーザーへのリンク**: 各カードの「特定ユーザーに設定」ボタンから、ユーザーID（UID）を指定して個別にメニューを切り替える機能。
- **エイリアス管理**: リッチメニュー・エイリアスの作成・紐付け管理。
- **既存メニューの複製**: 既存メニューの設定を引き継いで、画像だけ変えて新規作成する機能。
